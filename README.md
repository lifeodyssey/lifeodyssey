## 👋 About Me

I'm a backend-focused software engineer at **Thoughtworks**, working around backend systems, cloud infrastructure, DevOps automation, and practical AI engineering.

I care about systems that can survive real production pressure: clear boundaries, boring reliability, observable behavior, and workflows that help teams move faster without making the codebase harder to change.

Recently, I'm spending more energy on **Agenting AI**: coding agents, harness engineering, context engineering, agent orchestration, token budgeting, and practical workflows that bring AI into everyday software delivery.

- 🧱 **Backend:** APIs, service design, Spring Boot, FastAPI, PostgreSQL
- ☁️ **Cloud / Infra:** AWS, Terraform, Terragrunt, Docker, CI/CD
- 🤖 **Agenting AI:** coding agents, agent orchestration, context engineering, eval harnesses
- ✍️ **Writing:** long-form notes about AI engineering, backend, cloud, and developer tooling
- 🌱 **Principle:** build useful things, then make them maintainable

### Agenting AI Labs

| Project | What it explores |
|---|---|
| [Agent Patterns Lab](https://github.com/lifeodyssey/agent-patterns-lab) | Offline-first tutorial and reference implementation for agent design patterns, from function calling and structured output to ReAct, workflow loops, multi-agent systems, and eval harnesses. |
| [Craftsmanship Skills](https://github.com/lifeodyssey/craftsmanship-skills) | Agent skills distilled from Clean Code and Refactoring, bringing software craftsmanship into AI coding agents. |
| [OpenTeamConfig](https://github.com/lifeodyssey/opencode-team-config) | Team-level OpenCode setup for plugins, MCPs, agents, skills, hooks, TDD workflow, worktrees, safety rules, and shared project conventions. |
| [Runit](https://github.com/lifeodyssey/reins) | Harness engineering experiment for Claude Code, exploring structured sprint orchestration and plugin-based coding workflows. |
| [CodeFlow](https://github.com/lifeodyssey/codeflow) | Experimental multi-AI orchestration system exploring real-time feedback, provider routing, CLI integration, and Git worktree isolation. |
